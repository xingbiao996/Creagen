# 创谱 · Creagen Web 应用规范 (Spec)

## 产品概述 (Product Overview)
**创谱 · Creagen** 是一个旨在提取优秀创作者内容谱系，生成可学习的创作蓝图的 Web 应用。
- **Slogan**: 提取优秀创作者的内容谱系，生成可学习的创作蓝图。
- **目标**: 提取优秀创作者的视频内容，分析其创作规律，生成一张「创作配方卡」，明确告诉学习者“你应该怎么做”。

## 技术架构设计 (Technical Architecture)
- **前端**: React 18 + TypeScript + Vite + Arco Design (UI 组件库，作为核心且唯一的 UI 框架) + Less/CSS Modules (用于自定义样式) + Zustand (用于状态管理) + Lucide React (图标)。
- **后端**: Node.js + Express + TypeScript。
- **数据存储**: SQLite / PostgreSQL，用于存储已生成的创作配方卡及创作者信息。
- **AI 引擎**: 接入大语言模型 (LLM)，处理从视频中提取的元数据。
- **辅助工具**: Tampermonkey (油猴) 脚本，提供抖音精选网页端的快捷提取与基础分析浮窗功能。
- **UI/UX 规范**:
  - **主题与 VI 设计**: 确立品牌整体 VI 规范，默认主题色设定为 `#66ccff`，并支持主题色自定义。页面设计必须精美且具现代感。
  - **排版规范**: 在中文语境中，界面 UI 及文案中的引号必须严格规范为单直角引号「」和内嵌双直角引号『』，以替代传统的双引号“”和单引号‘’；在西文语境则使用标准的西文标点。

## 为什么 (Why)
新手创作者在学习优秀视频时往往只能看到表象，缺乏系统性的拆解框架。通过自动化提取视频数据并运用 AI 进行结构化分析，可以为用户提供一套清晰的创作配方，降低学习门槛，提升创作质量。

## 更改内容 (What Changes)
- 初始化全栈 Web 应用项目结构，前端集成 Arco Design 设计体系。
- 构建前端现代化精美界面，包含主页（推荐抖音精选优秀创作者、视频及已生成的创作配方卡）、URL 输入分析区域、配方卡展示页、历史记录页。
- 确立全站 VI 视觉规范与严格的中文字体标点排版规范（使用「」和『』）。
- 构建后端 API 服务，支持视频信息抓取（参考 `douyin_parse` 获取实际视频地址）、结合上下文的 AI 分析编排、数据库持久化。
- 开发油猴脚本扩展，针对抖音精选网页版提供浮窗基本分析功能及一键提交至系统的入口。

## 影响 (Impact)
- 影响的规范：建立全新的前端 UI/UX 与排版规范（Arco Design、标点符号要求）。
- 影响的代码：创建全新的前端代码库、后端服务代码库和浏览器扩展脚本。

## 新增需求 (ADDED Requirements)
### 需求 1：多渠道视频输入与主页推荐
主页需展示推荐的抖音精选优秀创作者、视频以及他们已生成的创作配方卡。同时，系统必须提供统一的视频 URL 输入入口（接收手动输入或油猴脚本传递的抖音精选网页 URL），并解析视频的实际地址、标题、简介及部分评论。
#### 场景：浏览主页与输入链接
- **当** 用户访问主页时
- **则** 看到精心设计的推荐内容（创作者、热门视频、优秀「创作配方卡」），并能通过输入框提交新链接。

### 需求 2：上下文感知的 AI 创作配方卡分析
AI 引擎必须基于新视频的标题、简介、部分评论，**以及该同创作者之前已分析生成的已有「创作配方卡」作为上下文**，生成全新的 7 维度配方卡。必须逐项输出：
1. **内容定位**——该类内容的核心价值主张
2. **叙事结构**——视频开场、发展、高潮、结尾的规律
3. **选题偏好**——常做哪些类型选题，有什么规律
4. **视觉风格**——画面色调、字幕风格、镜头语言
5. **情绪节奏**——视频的情绪起伏曲线
6. **差异化特征**——与其他同类创作者最不同的地方
7. **观众互动模式**——与观众的互动方式
以及附加的 **3-5条可执行行动建议**。
#### 场景：完成 AI 分析
- **当** 视频数据解析完毕并提交至 AI 模型时
- **则** 系统展示加载动画，随后渲染出结构化的 7 维度创作配方卡和行动建议。

### 需求 3：数据持久化与历史管理
生成的创作配方卡必须存储在服务端供以后查看。
#### 场景：查阅历史配方卡
- **当** 用户进入历史记录页面或再次分析同一创作者的视频时
- **则** 系统能从数据库中读取并展示相关的历史配方卡信息。

### 需求 4：系统设置与 OpenAPI 接入
系统需要提供灵活的配置能力和标准化的接口文档。
#### 场景：自定义 AI 模型与查阅接口
- **当** 用户希望使用自己的大模型 API 时
- **则** 可以在「设置」页面配置 OpenAI 兼容的 Base URL、API Key 和模型名称，应用将使用该配置进行后续分析。
- **当** 开发者需要接入或查阅系统接口时
- **则** 可以访问后端提供的 OpenAPI (Swagger) 接口文档，快速了解和测试 API。
